Проект 1: Анализ сетевого трафика (Wireshark)

## Цель

Захват и анализ реального сетевого трафика.

---

## Шаги

* Запущен Wireshark на интерфейсе `eth0`
* Сгенерирован трафик через браузер
* Применены фильтры: `dns`, `http`, `tls`

---

## Результаты

### DNS

Обнаружены запросы к доменам:

* `google-analytics.com`
* `consent.cookiebot.com`

---

### HTTP / OCSP

* Зафиксированы OCSP-запросы
* Используются для проверки сертификатов

---

### TLS

* Основной трафик зашифрован
* Обнаружены пакеты TLSv1.3 (Application Data)

---

## Вывод

Сетевой трафик включает:

* DNS-запросы
* проверку сертификатов (OCSP)
* зашифрованное HTTPS-соединение

Подозрительной активности не обнаружено.

# Проект 3: Анализ вредоносного трафика (Malware Traffic Analysis)

## 📌 Цель

Выявление подозрительной активности в сетевом трафике и определение возможного поведения вредоносного ПО.

---

## Используемые инструменты

* **Wireshark**
* **Kali Linux**
* Анализируемый файл:
  `2026-02-28-traffic-analysis-exercise.pcap, https://malware-traffic-analysis.net/2026/02/28/index.html`

---

## Предполагаемая жертва

* IP-адрес: `10.2.28.88`

Узел инициирует сетевые соединения и взаимодействует с внешними ресурсами.

---

## Подозрительная инфраструктура

### C2-сервер

* IP-адрес: `45.131.214.85`
* Endpoint: `/fakeurl.htm`

---

## Ход анализа

### 1. Анализ сетевых соединений

* Использован раздел **Statistics → Conversations**
* Выявлен IP с наибольшим объёмом трафика — `45.131.214.85`

---

### 2. Анализ DNS-трафика

* Применён фильтр: `dns`
* Обнаружены стандартные домены (Google, сервисы аналитики)
* Подозрительных доменных имён не выявлено

Вредоносная активность использует прямой IP (обход DNS)

---

### 3. Анализ HTTP-трафика

* Применён фильтр: `http`
* Обнаружены множественные **POST-запросы** к:

  ```
  http://45.131.214.85/fakeurl.htm
  ```

---

### 4. Поведенческий анализ

Выявлены признаки:

* Повторяющиеся POST-запросы
* Обращение к одному и тому же IP
* Отсутствие доменного имени
* Регулярность соединений

Указывает на автоматизированную активность

---

### 5. Анализ содержимого (Follow TCP Stream)

* Проанализированы TCP-сессии
* Выявлена передача данных в теле POST-запросов

Возможна передача:

* системной информации
* пользовательских данных

---

## Индикаторы компрометации (IOC)

### IP-адреса

* `45.131.214.85`

---

### Сетевое поведение

* Повторяющиеся HTTP POST-запросы
* Связь с внешним IP без DNS
* Регулярная передача данных

---

## Классификация вредоносного ПО

### Тип: Infostealer (предположительно)

---

### Обоснование:

* Использование POST-запросов для передачи данных
* Наличие C2-коммуникации
* Отсутствие загрузки файлов
* Поведение соответствует утечке данных

---

## Соответствие MITRE ATT&CK

* **T1071.001** — использование веб-протоколов (HTTP)
* **T1041** — эксфильтрация данных через C2
* **T1005** — сбор данных с локальной системы

---

## Итоговый вывод

Узел `10.2.28.88` демонстрирует поведение, характерное для заражения вредоносным ПО типа infostealer.

Обнаружены признаки:

* связи с командным сервером (C2)
* регулярной передачи данных
* возможной утечки информации
